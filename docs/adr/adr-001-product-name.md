# ADR-001: Product Name

**Date:** 2026-01-01
**Status:** Accepted
**Deciders:** CEO, Ilon (CTO)

## Context

Need a memorable, available name that communicates the product's value proposition.

## Options Considered

1. **Faultbox** — Sandbox for faults, memorable
2. **Invariant** — FM concept, sounds enterprise
3. **Whatif.dev** — Clear value prop, conversational
4. **Clearpath** — Understanding focus
5. **Simstorm** — Simulation focus

## Decision

**Faultbox**

## Rationale

- Immediately communicates value (fault injection, sandbox)
- Memorable and easy to say
- Works for both startup and enterprise positioning
- Domain likely available (faultbox.io)
- No direct competitors using this name

## Consequences

- Need to secure domain and social handles
- Name works for all product phases (CLI, Desktop, SaaS)
