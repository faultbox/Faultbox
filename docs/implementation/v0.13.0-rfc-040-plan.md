# v0.13.0 RFC-040 Determinism Levels — Implementation Plan

> **Status:** Draft, 2026-05-08. Branch: `epic/v0.13.0-determinism` (worktree at `.claude/worktrees/v0.13.0-determinism`). Tracking issue: [#109](https://github.com/faultbox/Faultbox/issues/109). Roadmap companion: [#114 RFC-046](https://github.com/faultbox/Faultbox/issues/114).
>
> **Cadence:** Single rc cycle. No real users yet (2026-05-08), so we don't ramp `unmediated_io` events through a "warnings now, enforced later" pre-strict phase — strict mode is on from day one. Land detection + escape hatches + strict mode + docs + tests in one rc.

## Scope recap

Section 8 of RFC-040 commits to six chunks for v0.13.0:
- **8.1** Detection of `unmediated_io` events with category metadata
- **8.2** Declarative escape hatches (`determinism()` builtin + `nondeterministic_ok=`)
- **8.3** Strict mode (default `strict=True`, fails on unmediated category not in allow-set)
- **8.4** Reserved syntax (`level="L0"|"L1"`, `runtime="default"` only)
- **8.5** Documentation
- **8.6** Tests + #84 coverage gate compliance

The five categories: `clock`, `rand`, `dns`, `network-unmediated`, `fs-unmediated`.

## Phase 2 — Starlark surface
### 2.1 Top-level `determinism()` builtin

In `internal/star/builtins.go`, register at the table starting `runtime.go:builtins()` line 28:

```go
"determinism": starlark.NewBuiltin("determinism", rt.builtinDeterminism),
```

Implementation pattern mirrors `builtinNondet` (line 1753) and `builtinService` (line 157):

```go
func (rt *Runtime) builtinDeterminism(thread *starlark.Thread, fn *starlark.Builtin, args starlark.Tuple, kwargs []starlark.Tuple) (starlark.Value, error) {
    if rt.detExplicit {
        return nil, fmt.Errorf("determinism(): may only be called once per spec")
    }
    var (
        level  = "L1"
        rtName = "default"
        strict starlark.Value = starlark.None
        allowV starlark.Value = starlark.None
    )
    if err := starlark.UnpackArgs("determinism", args, kwargs,
        "level?", &level,
        "runtime?", &rtName,
        "strict?", &strict,
        "allow?", &allowV,
    ); err != nil {
        return nil, err
    }

    // 8.4 reserved-syntax gating.
    switch level {
    case "L0", "L1":
        // ok
    case "L2", "L3", "L4", "L5":
        return nil, fmt.Errorf("determinism(level=%q): reserved for a future Faultbox release; v0.13.0 supports only L0 and L1", level)
    default:
        return nil, fmt.Errorf("determinism(level=%q): must be one of L0, L1 (L2..L5 reserved)", level)
    }
    switch rtName {
    case "default":
        // ok
    case "gvisor":
        return nil, fmt.Errorf("determinism(runtime=%q): reserved for a future Faultbox release; v0.13.0 supports only runtime=\"default\"", rtName)
    default:
        return nil, fmt.Errorf("determinism(runtime=%q): must be \"default\" (\"gvisor\" reserved)", rtName)
    }

    // strict semantics — at L0, reject; at L1, default true.
    if level == "L0" && strict != starlark.None {
        return nil, fmt.Errorf("determinism(strict=...): has no effect at L0 (no violation events exist); remove or change level")
    }
    strictBool := true
    if b, ok := strict.(starlark.Bool); ok {
        strictBool = bool(b)
    }

    // allow validation against KnownCategories.
    allow := make(map[string]bool)
    if list, ok := allowV.(*starlark.List); ok {
        iter := list.Iterate()
        defer iter.Done()
        var v starlark.Value
        for iter.Next(&v) {
            s, ok := starlark.AsString(v)
            if !ok {
                return nil, fmt.Errorf("determinism(allow=...): list items must be strings (categories), got %s", v.Type())
            }
            if !isKnownCategory(s) {
                return nil, fmt.Errorf("determinism(allow=...): unknown category %q (known: %s)", s, strings.Join(KnownCategories, ", "))
            }
            allow[s] = true
        }
    } else if allowV != starlark.None {
        return nil, fmt.Errorf("determinism(allow=...): must be a list of category strings")
    }

    rt.detLevel = level
    rt.detRuntime = rtName
    rt.detStrict = strictBool
    rt.detAllow = allow
    rt.detExplicit = true
    return starlark.None, nil
}
```

Error-message text matches the RFC verbatim where the RFC quotes a string (e.g. *"determinism level X is reserved for a future Faultbox release"*). The exact phrasing should be lifted into a single `fmt.Errorf` per gating branch so customer-facing strings match the RFC.

## Phase 3 — Detection
Detection is the load-bearing piece. Each category needs a concrete emit site.

### 3.1 `clock` and `rand` — seccomp filter additions

Both `clock_gettime` (228 amd64 / 113 arm64) and `getrandom` (318 amd64 / 278 arm64) are syscalls already in the name table. The challenge: the filter only intercepts syscalls referenced in `requiredSyscalls()` (`runtime.go:2493`). At L1 with strict mode active, we must add these unconditionally for any service whose effective allow-set does *not* already include `clock` and `rand` — because if it does, there's no point intercepting just to throw the event away.

Implementation in `runtime.go:requiredSyscallsForService` (line 2570):

```go
// L1 detection: add clock/rand interception unless tolerated.
if rt.detLevel == "L1" {
    eff := rt.effectiveAllow(svcName)
    if !eff[CategoryClock] {
        found["clock_gettime"] = true
    }
    if !eff[CategoryRand] {
        found["getrandom"] = true
    }
    if !eff[CategoryNetworkUnmediated] {
        found["connect"] = true   // see 3.3
    }
}
```

A new helper `Runtime.effectiveAllow(svcName)` merges spec-level + per-service:
```go
func (rt *Runtime) effectiveAllow(svcName string) map[string]bool {
    out := make(map[string]bool, len(rt.detAllow))
    for k := range rt.detAllow { out[k] = true }
    if svc, ok := rt.services[svcName]; ok {
        for k := range svc.NondeterministicOK { out[k] = true }
    }
    return out
}
```

VDSO caveat: Go's `time.Now()` and many libc paths use VDSO (`__vdso_clock_gettime`) which seccomp cannot intercept. The RFC explicitly says "best-effort; VDSO unreachable". We should:

- Document this in `docs/determinism.md` (Phase 6).
- Avoid surprising customers by emitting a single one-time per-service warning event (`unmediated_io_caveat`) at service-start when `clock` is being detected, noting that VDSO calls won't be caught. This sets honest expectations.

Now wire the emit. In `internal/star/runtime.go:1519` (`makeSyscallCallback`), extend the callback:

```go
return func(evt engine.SyscallEvent) {
    // existing emit at runtime.go:1539
    rt.events.Emit("syscall", svcName, fields)

    // NEW: unmediated_io detection at L1.
    rt.detectUnmediatedSyscall(svcName, evt)

    if (evt.Syscall == "connect" || evt.Syscall == "sendto") &&
        strings.HasPrefix(evt.Decision, "allow") {
        rt.mergeClocksForNetworkCall(svcName)
    }
}
```

`detectUnmediatedSyscall` is new in `internal/star/determinism.go`:

```go
func (rt *Runtime) detectUnmediatedSyscall(svcName string, evt engine.SyscallEvent) {
    if rt.detLevel != "L1" {
        return
    }
    var category string
    switch evt.Syscall {
    case "clock_gettime":
        category = CategoryClock
    case "getrandom":
        category = CategoryRand
    case "connect":
        // Decided by 3.3 via DestAddr lookup.
        return
    default:
        return
    }
    rt.events.Emit("unmediated_io", svcName, map[string]string{
        "category": category,
        "syscall":  evt.Syscall,
        "pid":      fmt.Sprintf("%d", evt.PID),
    })
}
```

### 3.2 `dns` — DNS resolution detection

There is no first-class DNS resolver in Faultbox today (`grep` showed no `getaddrinfo`/`/etc/resolv.conf` handling outside test cert generation). The realistic L1 detection signal is:

**Rule:** any `connect()` whose destination port is `53` (UDP or TCP) and whose destination IP is NOT one of the proxy listen addresses or interface addresses. This catches DNS-over-the-wire calls the SUT makes to its real resolver.

Implementation: in the connect handler (Phase 3.3 below), if `port == 53`, set `category = CategoryDNS` (overriding `network-unmediated`). DNS-over-HTTPS/DoH is out of scope and acknowledged in docs.

### 3.3 `network-unmediated` — connect to undeclared addresses

This is where we extend the "did this connect match a proxied interface" signal. Today, `connect` is intercepted only when a fault rule targets it (`engine/launch_linux.go:409`). The path that reads sockaddr already exists: `seccomp.ReadSockaddrFromProcess` (`filter_linux.go:468`).

**Strategy:** at L1, install a `connect` interception unconditionally on every faulted service (already covered by the `requiredSyscallsForService` extension in 3.1). In `engine/launch_linux.go:handleNotification` (line 289), after the rule loop and before the default `seccomp.Allow`, add a "report-only" path that reads the destination, asks the runtime "is this address mediated?", and emits the event if not.

The Engine package cannot call back into `internal/star` directly (would invert the dependency). The clean wiring: extend the `OnSyscall` callback signature to optionally carry a destination address, and let `makeSyscallCallback` do the classification.

Concretely, modify `engine.SyscallEvent` (`internal/engine/session.go:29`) — add:

```go
DestIP   string `json:"dest_ip,omitempty"`
DestPort int    `json:"dest_port,omitempty"`
```

…and in `handleNotification` after the existing `seccomp.ReadSockaddrFromProcess` call (or read it if no rule did), populate these on the SyscallEvent before `emitSyscallEvent`. Doing this only for `syscallName == "connect"` is fine.

Then in `makeSyscallCallback`:

```go
func (rt *Runtime) detectUnmediatedNetwork(svcName string, evt engine.SyscallEvent) {
    if rt.detLevel != "L1" || evt.Syscall != "connect" || evt.DestIP == "" {
        return
    }
    if evt.DestPort == 53 {
        rt.emitUnmediatedIO(svcName, CategoryDNS, evt)
        return
    }
    if rt.isMediatedAddress(evt.DestIP, evt.DestPort) {
        return  // proxied interface — mediated
    }
    if isLocalhost(evt.DestIP) && rt.isProxyListenPort(evt.DestPort) {
        return  // SUT is talking to a Faultbox proxy
    }
    rt.emitUnmediatedIO(svcName, CategoryNetworkUnmediated, evt)
}

func (rt *Runtime) isMediatedAddress(ip string, port int) bool {
    // Check declared interface ports; for container mode, also HostPort.
    for _, svc := range rt.services {
        for _, iface := range svc.Interfaces {
            if iface.Port == port || iface.HostPort == port {
                return true
            }
        }
    }
    return false
}
```

A separate helper `isProxyListenPort` consults the `proxy.Manager`. It needs a new method on `proxy.Manager`:

```go
// in internal/proxy/proxy.go
func (m *Manager) IsListenPort(port int) bool {
    m.mu.Lock()
    defer m.mu.Unlock()
    for _, rp := range m.proxies {
        if portOf(rp.listenAddr) == port {
            return true
        }
    }
    return false
}
```

### 3.4 `fs-unmediated` — defer to rc2 or beyond

The RFC labels this best-effort and lowest-priority. The natural detection point would be `openat` paths outside container-mediated mounts, but mediated paths in v0.12.x are ill-defined (volumes= is not a deny-list, observe= file-tail is read-only). The cleanest move:

- **rc1**: ship the category constant (`CategoryFSUnmediated`) so spec authors can put it in `allow=`/`nondeterministic_ok=` without errors, but emit zero events for it. The string is reserved.
- **rc2**: a follow-up issue tracks the actual detection (an `openat` to a path outside `svc.Volumes` mounts and outside `IsSystemPath`).

This is consistent with the RFC's "best-effort" framing and lets us land rc1 on time.

```go
            Matrix:     matrixInfo,

```

`strictEffective()` and `firstStrictViolation` go in `internal/star/determinism.go`:

```go
func (rt *Runtime) strictEffective() bool {
    if rt.detStrictOverride != nil {
        return *rt.detStrictOverride
    }
    return rt.detLevel == "L1" && rt.detStrict
}

func (rt *Runtime) firstStrictViolation(events []Event) *Event {
    for i := range events {
        ev := &events[i]
        if ev.Type != "unmediated_io" {
            continue
        }
        cat := ev.Fields["category"]
        eff := rt.effectiveAllow(ev.Service)
        if eff[cat] {
            continue
        }
        return ev
    }
    return nil
}
```

This is "first-violation-wins" — clearer error than "found N violations across services" and matches the RFC wording: *"strict mode fails the test on the first unmediated_io event"*.

No user base means no migration cushion is required — strict mode is on from day one. Suggested PR sequence inside the epic branch:

1. **PR 1 — data model & builtin (Phases 1+2).** New `internal/star/determinism.go`, `Runtime` fields, `ServiceDef.NondeterministicOK`, `determinism()` builtin, `service(nondeterministic_ok=)` kwarg. All parse-time tests in `internal/star/builtins_determinism_test.go`. Builtin works but no events flow yet — strict has nothing to fail on.
2. **PR 2 — detection (Phase 3).** Seccomp filter additions for `clock_gettime` / `getrandom` / `connect`. `engine.SyscallEvent` extended with `DestIP/DestPort`. `proxy.Manager.IsListenPort`. `detectUnmediatedSyscall` and `detectUnmediatedNetwork` in the `OnSyscall` callback. `unmediated_io` events appear in event log + bundle + report. Unit tests for helpers and golden-style integration test for one category.
3. **PR 3 — strict-mode wiring (Phase 4).** `strictEffective()` / `firstStrictViolation()` + insertion into `RunTest` + `StrictDeterminismViolation` outcome label flowing through bundle/manifest/report. Strict mode fails tests by default. Unit-level coverage in `builtins_determinism_test.go`.

3b. **PR 3 follow-up — Linux-side end-to-end goldens.** `poc/leaker` HTTP harness drives the four RFC scenarios + a tolerated variant. Goldens land in `testops/goldens/determinism_*.norm` and run via Lima on macOS. Required two product tweaks: (a) detection now installs the seccomp filter for tolerated categories too — tolerance suppresses the failure decision, not visibility; (b) `NormalizeTrace` filters `allow` syscalls to Go-runtime fds (eventfd / pipe / socket / /dev/null) to keep goldens deterministic across runs.
4. **PR 4 — CLI flag (Phase 5).** `--strict-determinism[=true|false]` flag + `RunConfig.StrictDeterminism` plumbing. CLI tests. Doc nit: fix RFC §8.3 wording to match the resolved open-questions log (CLI override is bidirectional).
5. **PR 5 — docs (Phase 6).** Tutorial chapter, `docs/spec-language.md` updates, feature-manifest rows, README bullet, CHANGELOG. Commit the previously-uncommitted RFC drafts (040, 041, 042, 043, 044, 046) here so the docs PR carries the full v0.13.0 paper trail.

Each PR builds on the previous; PR 3 is the visible behavior change for any spec on the branch.

## Risks and open hooks

1. **VDSO blindness for clock**: Go's `time.Now()` uses VDSO and seccomp cannot intercept it. The `clock` category will under-report for Go SUTs. The RFC anticipates this with "best-effort". Action: emit a one-shot `unmediated_io_caveat` event per service at startup so the bundle / report make the limit visible, and document in `docs/determinism.md`.

2. **`connect` interception cost**: today `connect` is intercepted only when a fault rule explicitly targets it. With L1 detection on, every connect on every faulted service goes through the notification loop. For high-connect-rate SUTs the per-service `nondeterministic_ok=["network-unmediated"]` opt-out is the documented escape.

3. **DNS port-53 heuristic**: misses DoH/DoT and over-reports for any service legitimately speaking DNS (a CoreDNS under test). Accepted trade-off; document that DNS-server SUTs should `nondeterministic_ok=["dns"]`.

4. **Per-service syscall surface drift**: `requiredSyscallsForService` (`runtime.go:2570`) is text-scan based. L1 detection adds three syscalls (`clock_gettime`, `getrandom`, `connect`) unless tolerated. A service that previously had no fault rules — and thus no seccomp filter — now gets one. Startup-latency check: run an existing recipe spec on the branch and compare wall-clock against main.

5. **Read-sockaddr placement**: `handleNotification` (`launch_linux.go:410`) reads sockaddr inside the rule loop today. To populate `DestIP/DestPort` for the always-on connect path, the read moves to the front of the function for `connect` syscalls. TOCTOU profile is unchanged from the existing rule-driven read.

6. **`fs-unmediated` reservation only**: the category is accepted in `allow=` / `nondeterministic_ok=` but emits zero events in v0.13.0. Document as "reserved, no detection yet" so specs setting it aren't surprised when detection lands. Refusing the kwarg now would force a spec edit later; reservation is the right move.

7. **Event volume from chatty SUTs**: a non-VDSO `clock_gettime` loop could emit thousands of `unmediated_io` events/sec and balloon the bundle. Strict mode's "fail on first violation" protects the test outcome but not the artifact size. If it materializes in real specs, rate-limit emission to ~100/sec/service with a single end-of-test "events suppressed" summary. Skip in v0.13.0 unless a golden exposes it.
